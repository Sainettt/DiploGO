# NestJS API & Prisma

Эта директория (когда будет создана `api`) содержит RESTful (или GraphQL) API для взаимодействия мобильного клиента с базой данных сервера. Приложение написано на TypeScript c использованием фреймворка NestJS.

## Стек
- **NestJS** — архитектура и контроллеры.
- **Prisma** — ORM для взаимодействия с БД и генерации типов.
- **PostgreSQL** — реляционная база данных для безопасного и надежного хранения данных.

## Структура модулей
Приложение NestJS будет разделено по логическим доменам (Modules):

1. **AuthModule**: Регистрация, логин (JWT).
2. **UsersModule**: Профили пользователей, управление статистикой и стриками (Streaks).
3. **TopicsModule**: Создание, удаление, редактирование тем.
4. **QuestionsModule**: CRUD для вопросов (с эндпоинтом для генерации вопросов с AI).
5. **GameSessionsModule**: Логика начала, проведения и завершения игровой сессии по конкретной теме.

## Prisma (ORM)
- Схема базы данных будет описана в `prisma/schema.prisma`.
- Модели: `User`, `Topic`, `Question`, `GameSession`, `Answer`.
- После любых изменений в схеме необходимо выполнить: `npx prisma migrate dev` (обновление БД) и `npx prisma generate` (обновление TypeScript типов клиента).

## Лучшие практики
1. **DTO (Data Transfer Objects)**: Используйте DTO вместе с `class-validator` и `class-transformer` для надежной валидации входящих данных в контроллерах.
2. **Dependency Injection**: Вся бизнес-логика должна находиться в провайдерах (Services) для упрощенного мокирования и модульного тестирования.
3. **Обработка ошибок**: Используйте встроенные средства фильтрации Exceptions от NestJS (`HttpExceptionFilters`).
